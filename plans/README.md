## Findings considered and rejected

- `BaseHead.astro` JSON-LD uses `set:html`, but the payload is produced with `JSON.stringify(...)` from structured objects, so it was not treated as an actionable XSS finding.
- `/api/trigger-build` bearer-token comparison itself was not treated as a vulnerability: the route has a production fail-fast when `CRON_SECRET` is missing and returns before reading the deploy hook on wrong tokens. The actionable gap is CI coverage, captured in Plan 004.
- Pagefind search was not reported as untested because `tests/playwright/blog-search.spec.ts` and `tests/playwright/pagefind-index.spec.ts` already cover search UI and build artifact generation.
- A full documentation sync from Astro 6 to Astro 7 was previously deferred; it is now covered by Plan 006.
- Local `.env` contains legacy Keystatic environment variable names. Because this repository is public and `.env` may contain sensitive local values, do not publish those locations in a public GitHub issue without explicit user confirmation. Plan 006 cleans `.env.example`; local `.env` cleanup/rotation should be handled privately if those values are real credentials.
