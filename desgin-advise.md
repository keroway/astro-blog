現状サイトは、テキスト構成としてはかなり良い方向です。
「keroway.com · yokohama」「vol. iii · software · yokohama」「横に渡って、技術を書く。」というコピー、Works / Blog / About の分け方、技術領域の整理は、ポートフォリオ兼技術ブログとして成立しています。現行サイトでも、Astro/Vercel、Yokohama、field notes という方向性が明確に出ています。 ￼

課題はおそらく、コンセプトはあるが、ビジュアル言語がまだ「記号の寄せ集め」に見えていることです。半円・逆三角・背景画像が悪いというより、サイト全体の意味体系と結びついていないため、装飾に見えてしまっている状態だと思います。

結論

おすすめは、いきなり全面リニューアルではなく、まずは次の方向で進めるのがよいです。

現行のタイポグラフィ・色・縦書き要素は残す。
そのうえで、HEROの背景と装飾を「東海道・戸塚・横浜・技術」の文脈に接続し直す。

具体的には、次のどれかが相性よいです。

1. 東海道の“道筋”を抽象化した線・地図・等高線・旅路モチーフ
2. 戸塚宿・東海道五十三次を参照した、現代的な版画風グラフィック
3. 横浜の港・近代建築・水面・都市スカイラインを抽象化した背景
4. ソフトウェアの構造、ログ、回路、地図を重ねた“field notes”系の意匠

一番「自分らしい」方向にしやすいのは、東海道の旅路 × 技術の記録 × 横浜在住エンジニアです。

推奨案A：東海道の道筋を抽象化する

戸塚は東海道の宿場町としての文脈が強く、横浜市の資料にも「東海道五拾三次之内 戸塚 元町別道」が紹介されています。絵の中には、茶屋、旅人、馬、橋、村落、遠景が描かれており、「移動」「記録」「中継点」という意味づけができます。

これをそのまま画像として使うのではなく、以下のように抽象化します。

- 画面左上から右下へ抜ける一本の道
- 細い罫線で描いた旧東海道のルート
- 戸塚・横浜・江戸・京都を思わせる小さな座標点
- 背景にごく薄い和紙テクスチャ
- HEROの右側にだけ、版画の構図を思わせる矩形フレーム

実装イメージとしては、SVGで十分です。
AI生成画像よりも、自作SVG + CSS変数の方がポートフォリオとしても技術者らしさが出ます。

例：

HERO背景:

- 極薄の紙色グラデーション
- 斜めに走る1本のパス
- パス上に小さな点
- 右上に薄い朱印風の “K”
- 下部に横浜の水面を思わせる水平線

半円や逆三角を使うなら、単なる図形ではなく、**「道標」「橋」「波」「朱印」「地図記号」**のような意味を与えるべきです。

推奨案B：生成AIで“版画風背景”を作る

生成AIを使う場合は、「完成絵」をそのまま貼るより、薄く処理して背景素材化するのがよいです。

プロンプトの方向性はこうです。

ポイントは、人・文字・細かい建物を避けることです。

- 彩度を落とす
- コントラストを落とす

- ノイズ・紙質だけ残す
- モバイルでは非表示または簡略版にする

という処理にします。

推奨案C：素材を探す

素材探しもありですが、個人サイトの独自性は弱くなります。使うなら、素材そのものよりも「加工前提」にした方がよいです。

探す素材は：

- 古地図
- 和紙テクスチャ

- 横浜港の水面
- 戸塚周辺の坂・道・橋・旧街道の写真
- 浮世絵のパブリックドメイン画像

ただし、浮世絵は出典・権利・利用条件を確認する必要があります。文化遺産オンラインにも戸塚の東海道五十三次資料がありますが、利用時は各所蔵元・掲載元の条件確認が必要です。

推奨案D：自分で撮影する

これはかなり良いです。

東戸塚・戸塚・旧東海道・坂道・橋・標識・川・駅周辺の写真を撮り、以下のように使います。

- 写真そのものは出さない

- 2値化する
- SVGトレースする
- ぼかして背景化する

「自分の現住所性」を入れるなら、生成AIよりも自分の写真をベースにした方が説得力が出ます。

3. UI/UX・a11yとの塩梅

デザイン性を上げたい場合でも、ポートフォリオ兼ブログでは以下を守るべきです。

優先順位

1. 本文の読みやすさ
2. 記事一覧への到達しやすさ
3. Works / About / Contact の明確さ
4. HEROの印象
5. 装飾

HEROが多少凝っていても、記事ページが読みにくいと技術ブログとして損をします。

縦書きの扱い

縦書きは良い特徴ですが、本文や主要導線には使わない方がよいです。

使ってよい場所：

- HEROの短いコピー
- セクションラベル
- 装飾的な地名
- 「戸塚」「横浜」「東海道」などの短語
- フッターの小さなサイン

避ける場所：

- ナビゲーション
- ボタン
- 記事タイトル一覧
- 長文
- モバイルでの主要情報

縦書きは「読み物」ではなく「印章・ラベル・空気感」として使うのが安全です。

色

現行の方向性を残すなら、色数は絞った方がよいです。

おすすめの色体系：

Base:

- sumi black / charcoal
- paper beige
- off white
  Primary:
- indigo / ai-iro
  Accent:
- vermilion / 朱
- muted gold または old green
  Functional:
- link blue は使いすぎない
- focus ring は明確に

朱色は多用すると一気に和風記念品っぽくなるので、5%以下がよいです。

使う場所：

- 小さな印
- active state
- hover underline
- tagの一部
- 外部リンクアイコン

フォント

現行の雰囲気なら、以下の組み合わせが無難です。

日本語

- 本文：Noto Sans JP / IBM Plex Sans JP / BIZ UDPGothic
- 見出し：Shippori Mincho / Noto Serif JP
- 縦書きアクセント：明朝系

英字・コード

- 英字UI：Inter / IBM Plex Sans
- コード：JetBrains Mono / IBM Plex Mono

技術ブログとしては、全文明朝寄りにしすぎると読みにくくなります。
おすすめは 本文ゴシック、見出しまたは装飾だけ明朝 です。

レイアウト

HOMEの構造は大きく変えなくてよいです。
ただし、HERO直下に「何のサイトか」をもう少し明確に出すとよいです。

現状のコピーは詩的で良いですが、初見には少し抽象的です。
補助文として、例えば：

Web, cloud, embedded systems, and tools — notes from hands-on software work.

または日本語で：

Web、クラウド、組み込み、開発ツールを横断しながら、試作・設計・運用の記録を残しています。

現行サイトにも「組み込みからクラウド、フロントエンドまで」という説明がありますが、HERO内での視認性を上げるとよいです。 ￼
