## ブログ記事のフィールド

### 基本情報

| フィールド | 用途 |
|---|---|
| タイトル | 記事タイトル。日本語可。URL slug とは独立管理。 |
| 概要 | 一覧・SEO・SNS で使う要約。1〜2 文を目安。 |
| 公開日 | 一覧の並び順と公開可否判定に使用。 |
| カテゴリ | 5 つの大分類から選ぶ。細かいトピックは tags に入れる。 |

### 本文・画像

| フィールド | 用途 |
|---|---|
| 本文 | Markdown 本文。本文中の画像は `public/images/blog` に保存される。 |
| ヒーロー画像 | カードサムネイル。16:9 推奨。 |
| OG 画像 | SNS 用のカスタム画像。未指定時は `/og/<slug>.png` が自動生成される。 |

### 公開・詳細設定

| フィールド | 用途 |
|---|---|
| 下書き | チェック時は本番公開されない。 |
| 更新日 | 大きな追記・修正があった場合のみ設定。 |
| 著者 | 通常は空でよい。空の場合はサイト既定の著者名を使う。 |
| Canonical URL | 外部掲載など正規 URL が別にある場合のみ設定。 |

## カテゴリ運用

カテゴリは「一覧での大分類・絞り込み」に使う。以下の 5 つに固定。

| 値 | 表示ラベル | 使い方 |
|---|---|---|
| `dev` | 開発・プログラミング | コード・アーキテクチャ・言語全般 |
| `hardware` | ハードウェア・電子工作 | Arduino / Raspberry Pi / 回路など |
| `tools` | ツール・インフラ | CLI・SaaS・クラウドサービスなど |
| `reading` | 読書 | 技術書・ビジネス書のレビュー |
| `event` | イベント・参加記 | Maker Faire 等の参加記録 |

---

## タグ運用

- タグは **既存タグの再利用を優先**する（新造は必要最低限に）。
- 大分類はカテゴリで、技術名・固有名詞・シリーズ名はタグで表現する。
- 例: `Rust`, `Astro`, `電子工作`, `zig` など。

## URL slug (ファイル名) について

Sveltia CMS は記事タイトルから slug（ファイル名）を自動生成します。  
**日本語タイトルの場合は slug が不適切な形になる可能性があります。**

新規作成時に slug を調整したい場合は、CMS の「ファイル名を変更」機能か、直接 Git でファイルをリネームしてください。

推奨形式: 英小文字・数字・ハイフンのみ（例: `astro-migration-guide`）

## Works エントリのフィールド

| フィールド | 用途 |
|---|---|
| プロジェクト名 | プロダクト名。日本語可。 |
| 概要 | 一覧・SEO・SNS 用要約（1〜2 文）。 |
| ステータス | active / wip / archived から選択。 |
| ランディングページ URL | 機能訴求・外部紹介ページ（必須）。 |
| リポジトリ URL | GitHub など実装の主参照（public のみ。private は設定しない）。 |
| デモ URL | 動作デモがある場合のみ。 |
| タグ | 技術スタック・カテゴリ（例: Rust, Astro, TypeScript）。 |
| 作成日 | プロジェクト開始日。 |
| 更新日 | 内容を大きく更新した場合のみ。 |
| ヒーロー画像 | カードサムネイル（16:9 推奨）。 |
| 注目プロジェクト | トップページの注目枠に表示するかどうか。 |

## ローカル編集のセットアップ

1. `pnpm run dev:astro` で Astro dev サーバーを起動
2. `http://localhost:4321/admin` にアクセス
3. **「ローカルリポジトリを使う」** を選択（File System Access API）
4. リポジトリのルートディレクトリを選択してアクセスを許可

> File System Access API は **Chromium ベースのブラウザ**（Chrome / Edge）が必要です。

## 本番 CMS へのアクセス

OAuth プロキシのセットアップ後、`https://keroway.com/admin` からアクセスできます。  
詳細なセットアップ手順は [cms-flow.md](./cms-flow.md) を参照してください。
