## コンテキスト

keroway.com の視覚アイデンティティは、葛飾北斎『富嶽三十六景』の「神奈川沖浪裏」一作に依拠している。

| 要素 | 出典 | 実装 |
|------|------|------|
| 紺碧・白浪・砂金のパレット | 「神奈川沖浪裏」の波・飛沫・空 | `src/styles/tokens.css`（`--kw-navy` / `--kw-paper` / `--kw-sand`） |
| 波紋モチーフ `.kw-wave` | 同上の波頭 | `global.css` の CSS `radial-gradient` 5 連弧。`Monogram.astro` のブランドシグネチャで使用 |

この「一作依存」には次の課題がある:

1. **モチーフの引き出しが波頭ひとつに限られる。** セクション区切り・ウォーターマーク・空白の演出など、波以外の装飾を足したくなったときに語彙がない。
2. **横浜という土地性を波の一語でしか語れていない。** keroway は横浜で記す個人サイト（縦書きレールのデフォルト文言「二〇二六年五月十三日 — 横浜にて記す」）であり、土地に根ざした視覚言語を広げる余地がある。

横浜近郊を描いた浮世絵には、波以外にも引用可能なモチーフがある。本 ADR は、これらを既存パレットと衝突しない形で**語彙として定義する**ことを目的とする。実装（CSS/SVG）は行わず、設計の合意と記録に範囲を限定する。

### 取り込みを検討するモチーフと出典

土地性を軸にモチーフを広げるにあたり、出典は単一の作者・シリーズに収まらない。**異なる作者・異なるシリーズをまたぐ**点を正確に押さえる。

| モチーフ | 画題 | 作者 | シリーズ |
|----------|------|------|----------|
| 波（既存） | 神奈川沖浪裏 | 葛飾北斎 | 『富嶽三十六景』 |
| 富士の遠景 | 東海道程ヶ谷 | 葛飾北斎 | 『富嶽三十六景』 |
| 松並木 | 東海道程ヶ谷 | 葛飾北斎 | 『富嶽三十六景』 |
| 宿場の街道風景（参照） | 戸塚・神奈川 | 歌川広重 | 『東海道五十三次』 |

「神奈川沖浪裏」「東海道程ヶ谷」は**葛飾北斎『富嶽三十六景』**、「戸塚」「神奈川」は**歌川広重『東海道五十三次』**の宿場である。両者は別の絵師・別のシリーズであり、安易に「すべて北斎」とまとめてはならない。

### 統合の軸

これらを統合する軸は「作者」ではなく **「横浜近郊という地理的近接」** である。程ヶ谷（保土ヶ谷）・戸塚・神奈川宿はいずれも東海道の横浜近郊の宿場であり、神奈川沖もまた横浜の海である。土地でくくることで、複数の絵師・シリーズの図像を一貫したサイトの語彙として無理なく束ねられる。

## 決定事項

**横浜近郊の浮世絵に由来する 3 つのモチーフ語彙（波 / 松並木 / 富士の遠景）を、地理的近接を統合軸として design-system.md に定義する。** 装飾の実装は本 ADR の範囲外とし、follow-up Issue で CSS/SVG により行う。

確定する内容:

1. **語彙の定義**: 波・松並木・富士の遠景それぞれに「意味」「用途指針」「想定トークン命名」「装飾実装の具体候補」を与える（詳細は design-system.md §9）。
2. **統合軸の明文化**: 出典が北斎『富嶽三十六景』と広重『東海道五十三次』をまたぐこと、束ねる軸が地理的近接（横浜近郊）であることを記録する。
3. **想定トークン命名**: 新色を導入する場合の命名規則を `--kw-pine-*`（松 = 緑系）・`--kw-fuji-*`（富士の遠景 = 霞んだ青/藍系）として提案する。**本 ADR・本 PR では `:root` への実トークン追加は行わない**（命名の予約のみ）。
4. **設計上の制約**: 既存パレット非衝突・ライト/ダーク両モード成立・`aria-hidden` での非露出・`prefers-reduced-motion` 尊重を全モチーフに適用する。

### ステータスを Accepted とする理由

本 ADR は「設計語彙を確定し、実装だけを follow-up に分離する」立場である。語彙の定義（design-system.md §9 への追記）は本 PR で実体化され、以降の装飾実装はこの定義に従う規範となる。設計判断そのものは保留せず確定するため、`Proposed`（提案・保留）ではなく `Accepted` が妥当と判断した。

実装（CSS/SVG・実トークン追加）が未着手である点は「決定の保留」ではなく「スコープの分割」であり、ステータスとは別問題として扱う（B1: 1 PR = 1 関心事 / B3: 差分分割の方針に準拠）。

## 検討した選択肢

| 選択肢 | 概要 |
|--------|------|
| **A: 横浜近郊の浮世絵を地理的近接で統合し語彙化** ★採用 | 波（既存）に松並木・富士の遠景を加え、「横浜近郊」を軸に複数絵師・シリーズの図像を一貫した語彙として定義 |
| **B: 北斎『富嶽三十六景』に限定して拡張** | 出典を北斎の同シリーズに絞り、「程ヶ谷」の松並木・富士のみ追加。広重の宿場要素は採らない |
| **C: 現状維持（波モチーフ一本）** | 「神奈川沖浪裏」依存を継続し、装飾は `.kw-wave` のみ |

## 比較評価

### A: 横浜近郊の浮世絵を地理的近接で統合（採用）

**メリット**:

- モチーフの引き出しが波・松並木・富士の遠景の 3 系統に広がり、セクション区切り・ウォーターマーク・空白演出など用途に応じて使い分けられる。
- 「横浜で記す」というサイトの土地性を、波の一語より厚く表現できる。
- 地理でくくるため、北斎・広重という別絵師の図像を無理なく一貫した語彙に束ねられる。将来さらに横浜近郊の図像を足す余地も残る。

**デメリット**:

- 出典が複数の作者・シリーズにまたがるため、由来の説明を正確に保つ運用負荷がある（本 ADR と design-system.md に統合軸を明記して対処）。
- 緑系・藍系の新色を導入すると、既存の紺碧/白浪/砂金の三色構成より管理対象が増える（彩度を抑え、実装は follow-up で慎重に検証して対処）。

### B: 北斎『富嶽三十六景』に限定して拡張

**メリット**:

- 出典が単一絵師・単一シリーズに収まり、由来の説明がシンプル。

**デメリット**:

- 「程ヶ谷」の松並木・富士は採れるが、土地性を広げる余地が北斎作品に縛られる。
- そもそも統合軸を「作者」に置くと、波（神奈川沖）と松並木（程ヶ谷）を結ぶ必然性が弱い。地理（横浜近郊）で束ねる方が自然であり、わざわざ作者で絞る利点が薄い。

**却下理由**: 統合軸を地理に置く案 A の方が拡張性・説明の一貫性ともに優れる。作者で絞ることに本質的な利点がない。

### C: 現状維持（波モチーフ一本）

**メリット**:

- 追加の設計・実装が一切不要。既存の `.kw-wave` のみで運用が完結する。

**デメリット**:

- 装飾の引き出しが波頭ひとつに固定され、セクション区切りやウォーターマークに使える語彙がない。
- 土地性の表現が頭打ちになる。

**却下理由**: 課題（モチーフの引き出し不足・土地性の頭打ち）を解消しない。Issue #176 の目的そのものを満たさない。

## 設計上の制約（実装時に follow-up Issue が遵守すること）

本 ADR で語彙を定義するが、実装は follow-up で行う。実装時に必ず守る制約を以下に固定する。これらは design-system.md §9 にも反映する。

1. **新フレームワーク禁止（implementation.md D1 / ADR 0001）**: 装飾は CSS `gradient` / `clip-path` / インライン SVG のみで描く。`.kw-wave`（`radial-gradient` のみで波紋を描く先例）と同じ「依存追加なし」の路線を踏襲する。
2. **既存パレット非衝突**: 紺碧（`--kw-navy`）・白浪（`--kw-paper`）・砂金（`--kw-sand`）の三色構成を壊さない。新色は彩度を抑え、既存トーンに馴染ませる（松 = 緑系 / 富士の遠景 = 霞んだ青・藍系）。
3. **ライト/ダーク両モード成立**: `[data-theme="dark"]` でも破綻しない。`currentColor` やセマンティックトークン経由で色を取り、両モードの値を tokens.css に定義する（実装時）。
4. **支援技術に非露出**: 装飾要素は `aria-hidden="true"` を付与する（`.kw-wave` と同じ扱い）。情報を持たない純粋な装飾である。
5. **モーション抑制の経路確保（implementation.md D4）**: モーションを足す場合は `@media (prefers-reduced-motion: reduce)` で必ずオフにできる経路を用意する。global.css には既に reduce 時に animation/transition を無効化する全称規則があるため、これに乗る形で実装する。
6. **想定トークン命名（予約のみ）**: 新トークンは `--kw-*` 命名規則（§4.2）に従い、`--kw-pine-*`（松）・`--kw-fuji-*`（富士の遠景）を予約する。**本 ADR では `:root` に実値を追加しない。**

## 出典の正確性に関する注記

本 ADR・design-system.md の記述で次を厳守する。誤記は土地性の物語そのものを損なうため、特に注意する。

1. **「神奈川沖浪裏」「東海道程ヶ谷」は葛飾北斎『富嶽三十六景』。** 程ヶ谷は「保土ヶ谷」とも表記される横浜の宿場で、松並木と富士の遠景が描かれる。
2. **「戸塚」「神奈川」は歌川広重『東海道五十三次』の宿場。** 北斎ではない。参照として街道風景の引用元に挙げるが、図像化の主対象は波・松並木・富士の遠景の 3 系統とする。
3. **統合軸は「作者」ではなく「横浜近郊という地理的近接」。** 「すべて北斎」と書かない。複数の絵師・シリーズを横断することを明記する。

## 結果

本 ADR の確定により、以下が実現できる:

- keroway の視覚言語が「神奈川沖浪裏」一作依存から、横浜近郊の浮世絵に根ざした 3 系統（波 / 松並木 / 富士の遠景）の語彙へ拡張される。
- 装飾を足す際に参照すべき語彙・命名・制約が design-system.md に集約され、実装 Issue がそれを規範として進められる。
- 出典の正確性（北斎と広重の別シリーズをまたぐこと・統合軸が地理であること）が記録され、由来の誤伝を防げる。
