# 0011 — モチーフ語彙に「道筋（東海道の街道）」を追加する

- **ステータス**: Accepted
- **決定日**: 2026-06-03
- **決定者**: @keroway

## コンテキスト

`desgin-advise.md` のデザイン助言を起点に、ホームの HERO 背景の刷新を検討した。助言の戦略（東海道・横浜・浮世絵の意味体系で視覚言語を統一する）は、既に [ADR 0007](./0007-motif-vocabulary-expansion.md) で「横浜近郊という地理的近接」を統合軸とするモチーフ語彙（波 / 松並木 / 富士の遠景）として採択済みである。

一方で、助言が HERO 背景に推す具体要素 —— **旧東海道の「道筋」を抽象化した一本の線・宿場を思わせる地点（点）・横浜の海を思わせる水面の水平線** —— は、現行の 3 モチーフのいずれにも対応しない。

| 現行モチーフ | 表すもの |
|------|------|
| 波 | 横浜の海・動・ブランド署名 |
| 松並木 | 街道沿いの連なり・区切り（静） |
| 富士の遠景 | 背景・奥行き・静謐 |

「街道そのもの（移動・記録・中継点）」を表す語彙が無い。戸塚宿・程ヶ谷宿は東海道の宿場であり、「旅人が宿場を通過しながら記録を残す」という構図は、keroway のサイトコンセプト（横浜で技術の手控えを残す）と強く響き合う。本 ADR は、この「道筋」を **4 つ目のモチーフ語彙として定義する**ことを目的とする。

ADR 0007 と同じく、本 ADR の範囲は語彙の定義に限定し、装飾の実装（HERO への適用）は follow-up Issue に分離する（B1: 1 PR = 1 関心事 / B3: 差分分割）。

### 統合の軸

道筋を束ねる軸は、既存 3 モチーフと同一の **「横浜近郊という地理的近接」** である。東海道の程ヶ谷（保土ヶ谷）・戸塚・神奈川宿はいずれも横浜近郊の宿場であり、それらを結ぶ街道の「道筋」は、波（神奈川沖）・松並木・富士の遠景（程ヶ谷）と同じ土地に根ざす。出典を新たな作者・シリーズに広げるものではなく、**既存の地理的物語に「街道」という連結線を一本足す**位置づけである。

## 決定事項

**横浜近郊の浮世絵語彙に「道筋（東海道の街道）」を 4 つ目のモチーフとして追加し、`docs/design-system.md §9` に定義する。** 統合軸は ADR 0007 と同じ地理的近接（横浜近郊）とする。装飾の実装は本 ADR の範囲外とし、follow-up Issue #242 で行う。

確定する内容:

1. **語彙の定義**: 道筋に「意味」「用途指針」「想定トークン命名」「装飾実装の具体候補」を与える（詳細は design-system.md §9.6）。
2. **三色構成の維持（新色を足さない）**: 道筋は新たな色トークンを導入せず、既存の `--kw-rule` / `--kw-fuji` / `--kw-sand` の範囲で描く。ADR 0007 の「既存パレット非衝突」をより強い形（新色追加そのものを行わない）で適用する。
3. **朱（vermilion）アクセント・朱印風 K の非採用**: 助言はこれらを推奨するが、本 ADR では採らない。紺碧・白浪・砂金の三色構成を維持する判断による。将来再検討する場合は別 ADR を起票する。
4. **トークン命名の予約のみ**: 道筋に専用色が必要になった場合に備え `--kw-road-*` を命名予約する。**本 ADR・実装 Issue とも、当面は既存トークンで賄い `:root` への実値追加は行わない**（ADR 0007 が `--kw-pine-*` / `--kw-fuji-*` を予約だけしたのと同じ扱い）。
5. **設計上の制約**: ADR 0007 §「設計上の制約」を全面的に継承する（依存追加なし・ライト/ダーク両対応・`aria-hidden` 非露出・`prefers-reduced-motion` 尊重）。

### ステータスを Accepted とする理由

ADR 0007 と同じ立場をとる。語彙の定義（design-system.md §9.6 への追記）は本 PR で実体化され、以降の装飾実装の規範となる。設計判断そのものは保留しないため `Proposed` ではなく `Accepted` が妥当。実装（CSS/SVG）未着手は「決定の保留」ではなく「スコープの分割」であり、ステータスとは別問題として扱う。

## 検討した選択肢

| 選択肢 | 概要 |
|--------|------|
| **A: 道筋を 4 つ目のモチーフとして追加（既存トークンで描く）** ★採用 | 街道の「道筋」を地理的近接の軸で語彙化。新色を足さず `--kw-rule` / `--kw-fuji` / `--kw-sand` で実装する |
| **B: 道筋を追加しつつ朱（vermilion）アクセント・朱印 K も導入** | 助言の全面採用。道筋に加え朱を 5% 以下のアクセントとして足し、朱印風 K を Monogram に導入する |
| **C: 現状維持（波 / 松並木 / 富士の 3 モチーフのまま）** | 街道の語彙を持たず、HERO 背景は富士ウォーターマーク一枚のまま |

---

## 比較評価

### A: 道筋を追加（既存トークンで描く）（採用）

**メリット**:

- 「街道＝移動・記録・中継点」というサイトコンセプトに直結する語彙を、土地性（横浜近郊の宿場）を保ったまま得られる。
- 新色を足さないため、紺碧・白浪・砂金の三色構成と管理対象を一切増やさない。ライト/ダーク両モードの色定義も既存トークンで完結する。
- 波・松並木・富士の遠景と座標・トーンを揃えやすく、HERO 背景でモチーフ同士が調和する。

**デメリット**:

- 既存トークンの範囲で描くため、道筋専用の色表現の自由度は低い（彩度・明度の微調整余地が小さい）。→ 必要が生じたら `--kw-road-*` の予約を実体化して対処する。

### B: 道筋 + 朱アクセント・朱印 K

**メリット**:

- 助言の意図を最も忠実に再現でき、朱印 K は強い視覚的アクセントになる。

**デメリット**:

- 朱（vermilion）は新色であり、ADR 0007 の「三色構成を壊さない」と正面から衝突する。朱は多用すると和風記念品的な印象に振れやすく、面積管理の運用負荷も生じる。
- ブランドの三色構成という確立済みの判断を変更するため、本来は独立した重い意思決定（別 ADR）を要する。道筋の追加に便乗させるべきではない。

**却下理由**: 三色構成の変更は道筋追加とは別関心事（B1）。今回は三色維持を選択し、朱は採らない。再検討は将来別 ADR で行う。

### C: 現状維持

**メリット**:

- 追加の設計・実装が不要。

**デメリット**:

- 街道の語彙を欠いたまま。HERO 背景を富士一枚から厚くする手立てがなく、Issue #241 の目的を満たさない。

**却下理由**: 課題（街道語彙の不在）を解消しない。

## 設計上の制約（実装 Issue が遵守すること）

ADR 0007 §「設計上の制約」を継承する。道筋に固有の点を加えて再掲する。

1. **新フレームワーク禁止（ADR 0001 / implementation.md D1）**: CSS `gradient` / `clip-path` / インライン SVG のみで描く。`.kw-wave` / `.kw-watermark-fuji` と同じ「依存追加なし」の路線。
2. **新色を足さない（本 ADR で強化）**: 道筋は `--kw-rule` / `--kw-rule-soft` / `--kw-rule-strong`（線）・`--kw-fuji`（遠景の道）・`--kw-sand`（地点のアクセント）の既存トークンで描く。`:root` への新色追加は行わない。
3. **ライト/ダーク両対応**: セマンティックトークン経由で色を取り、`[data-theme="dark"]` でも破綻しない（使用する既存トークンは両モード値を tokens.css に定義済み）。
4. **支援技術に非露出**: 装飾は `aria-hidden="true"` を付与する。
5. **モーション抑制**: 静止を基本とする。モーションを足す場合は `@media (prefers-reduced-motion: reduce)` でオフにできる経路を用意する（global.css の全称規則に乗せる）。
6. **可読性の保護**: HERO 背景に敷く際、前景テキスト・カードのコントラストを損なわない。富士の遠景と同じく「ごく薄いウォーターマーク」として扱う。

## 出典の正確性に関する注記

ADR 0007 の注記を継承する。

1. **道筋の参照元は東海道の街道風景**（程ヶ谷・戸塚・神奈川宿）。戸塚・神奈川は歌川広重『東海道五十三次』、程ヶ谷の松並木・富士は葛飾北斎『富嶽三十六景』であり、**別絵師・別シリーズをまたぐ**。
2. **統合軸は「作者」ではなく「横浜近郊という地理的近接」**。「すべて北斎」と書かない。道筋は新しい出典を足すのではなく、既存の地理的物語に街道という連結線を引く位置づけである。

## 結果

本 ADR の確定により、以下が実現できる:

- keroway の視覚語彙が、波 / 松並木 / 富士の遠景の 3 系統に「道筋（街道）」を加えた 4 系統へ拡張され、「移動・記録・中継点」というサイトコンセプトを背景装飾でも語れるようになる。
- 三色構成を維持したまま（新色なし）HERO 背景を富士一枚から厚くする実装方針が、design-system.md §9.6 と本 ADR の制約として固定される。
- 朱アクセント等の三色構成変更は別関心事として切り離され、必要時に別 ADR で扱う方針が記録される。
